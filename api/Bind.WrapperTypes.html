Enum WrapperTypes

Defines different types of parameter wrapper identifiers, which are used for hinting at how the method signatures should be generated.

Namespace: Bind
Assembly: cs.temp.dll.dll
Syntax
[Flags]
public enum WrapperTypes

Fields

Name Description
ArrayParameter

Function takes arrays as parameters - add ref/out and ([Out]) array overloads.
ArrayReturnType

Function returns a typed pointer - we have to copy the data to an array to protect it from the GC.
BoolParameter

Function takes bool parameter - C uses Int for bools, so we have to marshal.
ConvenienceArrayReturnType

Function normally returns an array via an out parameter. This overload returns a single item directly. e.g. void GenBuffers(int count, int[] ids) => int GenBuffer()
ConvenienceArrayType

Function normally takes an array in parameter. This overload takes a single item directly. e.g. void DeleteBuffers(int count, int[] ids) => DeleteBuffer(int id)
ConvenienceReturnType

Function normally returns a value via an out parameter. This overload returns a single item directly. e.g. void GetIntegerv(enum pname, out int value) => int GetInteger(enum pname)
GenericParameter

Function takes generic parameters - add ref/out generic and generic overloads.
GenericReturnType

Function returns a void pointer - maps to IntPtr, and the user has to manually marshal the type.
LegacyArrayParameter

Function takes a ref but we emit a legacy array overload to maintain backwards compatability.
None

No wrapper needed.
PointerParameter

Function that takes (in/ref/out) a naked pointer as a parameter - we pass an IntPtr.
ReferenceParameter

Function that takes a reference to a struct.
SizeParameter

Functions takes an IntPtr that corresponds to a size_t. Add an int32 overload for convenience.
StringArrayParameter

Function takes a String[] parameter
StringParameter

Function takes a String or StringBuilder parameter
StringReturnType

Function returns string - needs manual marshalling through IntPtr to prevent the managed GC from freeing memory allocated on the unmanaged side (e.g. glGetString).
UncheckedParameter

Function with bitmask parameters. Bitmask parameters map to UInt, but since we can only use signed types (for CLS compliance), we must add the unchecked keyword. Usually found in bitmasks
